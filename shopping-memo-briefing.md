# 買い物メモアプリ 企画まとめ

## 背景・動機

- BI畑出身、AIを使って何か作ってみたい（学習目的）
- 既存BIツールでできることより、「自分が実際に使えるもの」を作りたい
- ADHDのための買い物メモアプリというアイデアが浮かんだ

## コンセプト

**「常備品リストを見るだけで買い忘れに気づける」**

- 毎回ゼロからメモを書くのではなく、常備品プリセットをON/OFFするだけ
- リストを見るだけで「あ、醤油！」と気づけることが目的
- 管理を複雑化しないことが重要なコンセプト

## 確定機能

| 機能 | 備考 |
|------|------|
| 常備品プリセットリスト | 調味料・日用品など固定アイテムを登録 |
| ON/OFFトグル | 足りなくなったらONにして買うリストへ |
| パネル表示 | 横並びで縦長になりすぎないレイアウト |
| 並び替え | 店内動線などに合わせて順番を変えられる |
| ダークモード | 対応必須 |
| 画像添付 | ローカル保存・サムネイル圧縮。「夫が違うものを買ってくる問題」対策 |

## 見送った機能（理由付き）

| 機能 | 見送り理由 |
|------|-----------|
| 買い場所タグ | 管理複雑化 |
| ワンタップ完了・全部買ったボタン | 管理複雑化 |
| 在庫2段階ステータス | 管理複雑化 |
| 通知・リマインダー | 不要（通知無視が習慣化するリスクも） |
| 位置情報連動 | 不要 |
| 音声入力 | 不要 |
| 写真共有（他者との同期） | スタンドアロン方針と相反するため見送り |
| ウィジェット・バッジ表示 | 将来対応候補。現時点では個人用途なので不要 |
| ピンチアウト/INでパネル表示数変更 | 将来対応候補 |

## 技術方針

- **スタンドアロン**（サーバー通信なし）
- **Webアプリ（HTML単一ファイル）** としてまず作る
- データはブラウザの `localStorage` に保存
- 画像はbase64でローカル保存・サムネイル圧縮
- iPhoneのSafariで開き「ホーム画面に追加」でアプリ的に使用
- 配布方法候補：GitHub Pages / Netlify（固定URL・無料）

## 将来対応候補
